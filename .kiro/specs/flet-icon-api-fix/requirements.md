# Requirements Document: Flet Icon API Fix

## Introduction

Исправление использования устаревшего Flet API для иконок в компонентах `SegmentedButton` и `ft.Tab`. Проблема вызывала ошибку `AttributeError: 'Icons' object has no attribute '_set_attr_internal'` при попытке открыть модальные окна плановых транзакций и использовать фильтры по типам.

## Glossary

- **SegmentedButton**: Компонент Flet для выбора одного из нескольких вариантов с визуальным разделением
- **ft.Tab**: Компонент Flet для создания вкладок в интерфейсе
- **ft.Icons**: Класс Flet, содержащий константы иконок Material Design
- **ft.Icon**: Компонент Flet для отображения иконки

## Requirements

### Requirement 1: Исправление SegmentedButton

**User Story:** Как пользователь, я хочу открывать модальное окно создания плановой транзакции, чтобы добавлять новые плановые операции.

#### Acceptance Criteria

1. WHEN пользователь нажимает кнопку добавления плановой транзакции, THEN система SHALL открыть модальное окно без ошибок
2. WHEN модальное окно открывается, THEN система SHALL отобразить SegmentedButton с иконками для выбора типа транзакции
3. THE система SHALL использовать `ft.Icon(ft.Icons.XXX)` вместо прямого `ft.Icons.XXX` для иконок в SegmentedButton

### Requirement 2: Исправление ft.Tab

**User Story:** Как пользователь, я хочу использовать фильтры по типам транзакций, чтобы видеть только нужные мне данные.

#### Acceptance Criteria

1. WHEN пользователь переключает вкладки фильтров, THEN система SHALL отображать иконки без ошибок
2. THE система SHALL использовать `ft.Icon(ft.Icons.XXX)` вместо прямого `ft.Icons.XXX` для иконок в ft.Tab
3. WHEN пользователь открывает любой раздел с вкладками, THEN система SHALL корректно отображать все иконки

### Requirement 3: Совместимость с современным Flet API

**User Story:** Как разработчик, я хочу использовать актуальный Flet API, чтобы избежать проблем совместимости в будущем.

#### Acceptance Criteria

1. THE система SHALL использовать только актуальные методы Flet API (>= 0.25.0)
2. THE система SHALL обёртывать все иконки в `ft.Icon()` при передаче в компоненты
3. WHEN Flet обновляется, THEN система SHALL продолжать работать без изменений кода иконок
